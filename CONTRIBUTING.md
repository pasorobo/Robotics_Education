# Contribution Guide

このドキュメントは、Robotics Education Course リポジトリへの貢献ルールを定めます。インストラクター・学習者いずれも本ガイドラインに従ってください。

## ブランチ命名

ブランチ名は役割によって使い分けます（仕様書 §2.1 参照）。

| 役割 | 命名規則 | 例 |
|------|----------|----|
| インストラクター | `course/sp<N>-<topic>` | `course/sp1-foundation-week1` |
| 学習者 | `learner/<name>/wk<N>-<topic>` | `learner/taro/wk1-ros2-basics` |

## Commit prefix

コミットメッセージには以下のプレフィックスを使用します（仕様書 §2.1 参照）。

| プレフィックス | 用途 |
|----------------|------|
| `feat:`        | 新しいコンテンツ・機能の追加 |
| `docs:`        | ドキュメント（講義ノート・規約など）の追加・修正 |
| `lab:`         | Lab 演習ファイルの追加・修正 |
| `tool:`        | tools/ 配下のスクリプト・ツールの追加・修正 |
| `resource:`    | リファレンス・sandbox_reference 資料の追加・修正 |
| `chore:`       | ビルド設定・依存関係・CI など雑務 |
| `fix:`         | バグ修正・誤記訂正 |

## Commit author identity

このリポジトリのローカル git config は `user.name = pasorobo` / `user.email = goo.mail01@yahoo.com` に設定されています。特に指示がない限り上書きしないでください。

**重要:** コミットメッセージに `Co-Authored-By:` トレーラーを追加しないでください。Claude、Codex、AI、エージェント、コーディングツール等への言及もコミットメッセージに含めないでください。

## Pull Request

PR 説明には最低限以下を含めてください。

- **目的:** この PR で何を達成するか
- **スコープ:** 変更対象のファイル・ディレクトリ
- **検証コマンド:** レビュアーが動作確認に使うコマンド（例: `colcon build --packages-select <pkg>`）
- **関連仕様・計画リンク:** 対応する spec/plan ドキュメントのパスまたはセクション番号

## 原典（教育計画ドキュメント）の取り扱い

教育計画などの原典ドキュメントを移動・整理する際は以下に従います（仕様書 §3.1 #4 参照）。

- **すでに git 管理下にあるファイル:** `git mv <src> <dst>` を使用してください（履歴が保持されます）。
- **未追跡のファイル:** `mv <src> <dst>` で移動後、`git add <dst>` でステージングしてください。

## Sandbox 原則

`sandbox_reference/` は模擬データ・オフライン参考資料専用です（仕様書 §1.2 参照）。

- `.env`、認証情報（credentials）、ロボット接続情報などの秘密情報は**絶対に含めない**こと
- 実機ロボットへの接続情報・アクセスキーは**絶対に含めない**こと
- 使用するデータは mock データ / シミュレーションデータ / オフラインデータのみ

## Codex 利用ルール

Codex の利用は週によって段階的に拡大します（仕様書 §2.6 参照）。

- **Week 1:** 接続確認・ルール理解・prompt 5項目練習のみ。本格的なコード生成には使用しない。
- **Week 2 以降:** Lab 4b / 6b / 8b で本格運用開始。

## レビュー観点

PR レビュー時は以下の観点を確認します（仕様書 §2.6 参照）。

1. **diff サマリー:** 変更内容を簡潔に説明できるか
2. **動く根拠:** 動作する理由・テスト結果・実行ログを示しているか
3. **壊れうる条件:** 想定外の入力や環境で何が壊れうるかを把握しているか
4. **採用しない提案:** 検討して却下した代替案とその理由を記録しているか
5. **追加修正:** レビューコメントへの対応が完了しているか
